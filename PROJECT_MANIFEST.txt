================================================================================
THE 4 SERVICES
================================================================================

web (Nginx Alpine)
  - Port: 8080 (maps to 80 in container)
  - Image: Multi-stage (node:20 builder → nginx:1.27)
  - Purpose: Static frontend + proxy /api to api:3000
  - Healthcheck: GET / (HTTP 200)

api (Node.js 20 Alpine + Express)
  - Port: 3000 (exposed in dev, hidden in prod)
  - Dependencies: express, pg, redis, morgan
  - Routes:
    * GET /api/health → db/cache status
    * GET /api/products → list all (cached 30s)
    * GET /api/products/:id → fetch one (cached)
    * POST /api/products → create
    * GET /api/cache-stats → hit/miss stats
  - Runs as non-root user: node
  - Healthcheck: GET /api/health (JSON response)

db (PostgreSQL 16 Alpine)
  - Port: 5432 (not exposed to host)
  - Volume: pgdata (persistent)
  - Init: init.sql (CREATE TABLE + INSERT 10 rows)
  - Schema: products(id, name, price, stock, created_at)
  - User: trainer / trainshop_dev_only
  - Healthcheck: pg_isready

cache (Redis 7 Alpine)
  - Port: 6379 (not exposed to host)
  - Volume: cache_data
  - Purpose: Cache products (30s TTL) + track stats
  - Healthcheck: redis-cli ping
